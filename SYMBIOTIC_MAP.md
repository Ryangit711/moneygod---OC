## PART 4: THE ANTI-SYMBIOSES — Relationships That Should NOT Form

A reef also has predators and diseases. These relationships weaken the system:

| Anti-Symbiosis | Why It's Toxic |
|----------------|----------------|
| **Reading hierarchy-master WITHOUT live-data-workflow** | You know everything about plumbing. You never check it live. Information without action = ego without edge. |
| **Trading MES/MNQ WITHOUT plumbing-to-trade-bridge** | You execute setups without regime context. You take a breakout scalp on a day Net Liq is draining. The setup works statistically. Today is the exception. |
| **Buying a prop eval WITHOUT eval-mode-protocol.md** | You size like a funded trader on an eval. One bad day = account gone. The eval is not the same game as funded. Most people never learn this. |
| **Studying fill-your-cup WITHOUT trading-commandments.md** | You know where you want to go. You don't have the rules to survive the journey. The path to $10K/month runs through 10 drawdowns first. |
| **Reading CURRICULUM.md without doing it** | The most common anti-symbiosis. You know the order. You never execute. The curriculum is not a reading list — it's a prescription. |
| **Running GROWTH_PROTOCOL without adding new content** | You git push every session but never write new understanding. The system calcifies. You're maintaining a graveyard. |

**How to spot an anti-symbiosis:** You feel like you're working but nothing is changing. The reef looks healthy but no new coral is growing.

## PART 5: THE REEF'S LIFE CYCLE — One Complete Breath

```
START: You sit down. 3 hours ahead.

BREATH IN (Study):
  1. Open systems/3hr-daily-schedule.md
  2. Today is a TRADE day or STUDY day?
  3. If TRADE: systems/live-data-workflow.md → compute Net Liq → verdict
  4. If STUDY: pick a thread from FREEFALL_360.md or the hierarchy lexicon

FILTER (Bridge):
  5. systems/plumbing-to-trade-bridge.md → 4 decision trees
  6. Which instrument? Which edge? What size?
  7. trading/trading-commandments.md → read. Filter impulse.

EXHALE (Trade or Learn):
  8. If TRADE day:
     - trading/mes-mnq-playbook.md OR trading/complete-strategy-orb-eurusd.md
     - Execute. Journal. Post-session review.
  9. If STUDY day:
     - Pick a fuzzy term from hierarchy Part 11
     - Feynman it. Write your understanding.
     - Cross-reference with a core/.md file you haven't opened in 2 weeks.

RECYCLE (Growth):
  10. Did you learn something new? → Write it. Add to the right file.
  11. git add -A && git commit -m "[insight]" && git push
  12. The reef just grew one coral.

BREATH OUT: You stand up. The reef is alive. It will be here tomorrow.
```

## THE FINAL TRUTH

A symbiotic system doesn't need you to understand all of it at once. It needs you to understand ONE relationship at a time — and trust that the rest of the system supports that relationship.

- You don't need to know repo plumbing to run the live data workflow.
- But if repo stress spikes, the workflow will flag it, the bridge will tell you what it means, and you'll know why you shouldn't trade that day.
- And when you're curious what "repo" actually is, you open core/repo-plumbing.md.
- And the system rewards your curiosity with deeper understanding.
- Which makes next week's workflow more insightful.
- Which makes next month's trades more informed.

**That's the symbiosis. Not a curriculum you complete. A reef you inhabit.**

Every file needs every other file. Every session feeds every future session. No knowledge is wasted. No relationship is accidental.

You don't master the system. You live inside it.
